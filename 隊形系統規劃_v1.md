# 隊形系統規劃 v1（任務 #32）

> 由「固定羈絆」升級而來。核心：**預設多種隊形供玩家挑選**；每個隊形的佔位固定（像素座標），**每個站位各有加減成**，且**位置會影響戰鬥**（前排被集火、後排被保護）。新增獨立「隊形」入口，**每場戰鬥前皆可變更**。

---

## 1. 資料模型

每個隊形 = 3 個固定站位（對應戰士/遊俠/牧師），每站位含：
- `x, y`：像素座標（戰鬥畫面左半場）
- `row`：`front`／`mid`／`back`（決定被鎖定權重）
- 加減成：`atk / def / hp / heal`（可正可負）

```js
FORMATIONS = [
  { name:'標準陣', desc:'攻守均衡，戰士在前扛傷', slots:{
     warrior:{x:330,y:360,row:'front', def:+3},
     ranger :{x:200,y:255,row:'back',  atk:+4},
     priest :{x:200,y:440,row:'back',  heal:+5, hp:+8} } },

  { name:'鋒矢陣', desc:'雙前排猛攻，輸出高但脆', slots:{
     warrior:{x:340,y:300,row:'front', atk:+4},
     ranger :{x:340,y:420,row:'front', atk:+8, def:-2},
     priest :{x:190,y:360,row:'back',  heal:+6} } },

  { name:'鐵壁陣', desc:'重防禦持久，輸出較低', slots:{
     warrior:{x:300,y:360,row:'front', def:+6},
     ranger :{x:170,y:260,row:'back',  def:+3, atk:-2},
     priest :{x:170,y:460,row:'back',  heal:+4, def:+3} } },

  { name:'散獵陣', desc:'全員靠後風箏，傷害分散', slots:{
     warrior:{x:280,y:360,row:'mid',  def:+2},
     ranger :{x:160,y:300,row:'back', atk:+6},
     priest :{x:160,y:420,row:'back', heal:+5} } },
];
```

> 數值為初版，實作後再平衡。可隨日後新職業/第四人擴充站位。

---

## 2. 戰鬥影響（位置 = 戰術）

- **被鎖定權重**：`front` 高、`mid` 中、`back` 低。敵人選目標時依權重隨機 → 前排吃最多傷害、後排被保護。
  - 取代目前「敵人 60% 打戰士」的寫死邏輯，改成依隊形站位的權重。
  - 「散獵陣」無前排 → 敵人隨機平均分散（風箏感）。
- **站位加減成**：該站位的 atk/def/hp/heal 直接套到站此位的隊員（在 heroStat 計入，作法同原羈絆）。
- **戰鬥站位**：英雄的 x/y 直接用隊形座標（取代 Battle 寫死的 hpos）。

---

## 3. 與現有羈絆的關係

- 「羈絆升級為隊形」→ **以隊形站位加減成取代原本 3 條固定羈絆**的攻防加成。
- 「標準陣」刻意接近原羈絆數值，確保不選別的也不虧、行為連續。

---

## 4. UI／入口

- **公會大廳新增「⚔ 隊形」建築入口** → 隊形場景：
  - 左：隊形清單（4 種）可點選，顯示名稱與說明。
  - 右：所選隊形的**站位預覽**（小人按座標排出來）＋每站位的加減成標示。
  - 底：確認選用（存檔，跨輪保存 `GUILD.formation`）。
- **每場戰鬥前可變更**：在地城地圖（Map）加一個「⚔ 隊形」快速切換鈕，進同一場景或彈窗切換，回來不影響當前進度。

---

## 5. 實作步驟

1. data.js：`FORMATIONS` 資料。
2. state.js：`GUILD.formation`（存檔預設 0）、`formationSlot(sprite)`（回傳該隊員站位資料）；heroStat 改用隊形站位加減成（移除舊 bond 攻防）。
3. battle.js：英雄座標改用隊形座標；敵人選目標改為依 row 權重。
4. 新場景 FormationHall（隊形入口）＋ 公會大廳建築鈕。
5. Map 加「⚔ 隊形」快速切換。
6. 瀏覽器實測：選不同隊形 → 站位/加成/被集火行為改變，無錯誤。
