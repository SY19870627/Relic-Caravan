## 範圍與現況關鍵事證

- **`SKILLS` 從未定義**：`state.js`、`battle.menu.js` 都讀 `SKILLS[sprite]`，值恆為 `[]`。所以現行「升級三選一」抽到的池一直是空的——技能系統是空骨架。**本計畫＝第一次把它做出來，且直接做成規劃器。**
- **升級三選一流程**：`state.js gainXP()`（256–270）升級時 `RUN.pendingLevelups.push(i)` → `battle.menu.js showLevelups()`（33）彈窗三選一。**這條要拆掉。**
- **存檔模型**：`saveGuild()`（state.js 59）把整個 `GUILD` 序列化（隊伍免洗、本趟 level/技能不存）。**只要在 `defaultGuild()` 加欄位就會自動存**，`loadGuild()` 補一行防呆即可。
- **英雄組裝**：`buildHeroes()`（state.js ~345）由 `ROSTER[idx].skills`（名稱）＋`skillPlus` 查 `SKILLS[sprite]` 套 `skillWithPlus`。
- **戰鬥生成**：`battle.js` makeCombatant（~103）以 `sprite:h.sprite` 生我方 → **變身換 sprite 的切入點在這。**
- **大廳導航**：`guildhall.js` 用 `navCard(this,x,y,w,h,{...,onClick})` ＋ `this.scene.start('XxxHall')`；場景註冊在 `main.js` 的 `scene:[...]` 與 `index.html` 的 `<script>`。

## Stage 1 — 拆除三選一 ＋ 資料骨架

**目標**：移除三選一、補上 `SKILLS.warrior`、加持久狀態、升級時依配置表靜默發放。此階段不含 UI，可用「作弊：+資源」與既有存檔手動塞 plan 來驗證。

### 1-1 新增技能資料（`js/data.js`）
新增 `const SKILLS = { warrior:[...] }`，為**單一打平池**（不分支、不拆兩池），依《設計稿 v0.2》§4。每招物件欄位：
```
{ name, role:'ultimate'|'active'|'passive', type, cd?, uses?, mult?, frac?, amt?, dur?, tier(1|2|3), desc }
```
- `role:'ultimate'` ＝大招（變身熊人／牛頭人，2 個）；`'active'` ＝主動小招（5 個）；`'passive'` ＝被動（11 個）。
- 被動沿用引擎現有 type：`reflect / deathSave / lowHpDef / pierce / critVsStunned / lifesteal / startShield`。
- 大招與新主動/被動標新 type（Stage 3 實作 handler）：`transform / hyperarmor / cover / taunt / nuke / extraHit / execute / atkBuff` 及 AoE 版 `stun`。

### 1-2 持久狀態（`js/state.js`）
- `defaultGuild()`（27）新增兩欄：
  ```
  classPoints:{},   // 各職業點數上限：{warrior:Number}（活著回來累計）
  classPlans:{},    // 各職業配置：{warrior:{slots:[{act:'learn'|'upgrade', skill:'熊人變身'}, ... 長度 20]}}　← 無 branch 欄位
  ```
- `loadGuild()`（63）補防呆：`if(!GUILD.classPoints)GUILD.classPoints={}; if(!GUILD.classPlans)GUILD.classPlans={};`
- （`saveGuild()` 不必改——`GUILD` 整包已序列化。）

### 1-3 點數成本與預算工具（`js/state.js` 新函式）
- `skillCost(sprite, skillName)` → 依 tier 回 1/2/3。
- `upgradeCost(nthUpgrade)` → 回 n（第 n 次升級花 n 點）。
- `planSpent(sprite)` → 累加該 plan 已花點數。
- `planValid(sprite)` → 檢查 `planSpent <= classPoints[sprite]`，且 **大招 ≤ 1、主動小招 ≤ 3、被動不限**（依 `role` 計數）。

### 1-4 升級改「依表自動發放」（`js/state.js`）
- 新增 `grantPlannedSkill(idx, level)`：讀 `GUILD.classPlans[sprite].slots[level-1]`，對 `ROSTER[idx].skills / skillPlus` 套用（沿用 `skillWithPlus` 數學），**靜默**。
- `gainXP()`（256–270）：升級時改呼叫 `grantPlannedSkill(i, r.level)`，**移除 `RUN.pendingLevelups.push(i)`**（或保留陣列但不再塞，給一行小提示 toast）。

### 1-5 拆三選一彈窗（`js/scenes/battle.menu.js`）
- `showLevelups()`（33）改為直接 `this.advanceStep()`（不再彈窗）。
- 移除/停用 `levelCard / showReplace / _nextLevelup`（保留亦可，但不再被呼叫）。
- `rollLevelChoices / applyLevelChoice`（state.js 282/294）停用（規劃由 Stage 2 UI 接手）。

### 1-6 點數上限成長（`js/scenes/result.js`）
- 在「活著回來」（成功返回、非全滅）的結算處，對每個出戰職業：
  `GUILD.classPoints[sprite] = (GUILD.classPoints[sprite]||0) + 該趟最終等級;` 然後 `saveGuild()`。
- （實作時先確認 result.js 結算的確切函式與「是否全滅」判斷。）

**Stage 1 驗收**：手動在存檔塞一個 warrior plan（1 變身大招＋幾個小招/被動）→ 跑地城升級時，技能依表自動出現、無彈窗；活著回來後 `classPoints.warrior` 增加並存檔。

## Stage 2 — 職業規劃器 UI（`js/scenes/skilltreehall.js`，新檔）

仿 `formationhall.js / wagonhall.js` 的 hall 結構（`sceneBg / sceneHeader / panel / button / navCard`）。

**畫面元素**
- 職業切換（先只做戰士；之後沿用）。
- **無分支選擇**（已取消）。改為三類槽位：**大招 ×1**（變身二選一：熊人/牛頭人）、**主動小招 ×3**、**被動（不限）**。
- Lv1–20 配置列：每列一個下拉/點選 → 該級「習得某招」或「升級某招＋1」。
- 即時狀態列：已花點數 / 點數上限、大招 X/1、主動小招 X/3、被動數量、超出時該列標紅且 `planValid=false`。
- 「重置點數」鈕：清空該職業 plan（點數上限保留）。
- 寫入：操作即更新 `GUILD.classPlans[sprite]`，離開或存檔時 `saveGuild()`。

**接線**
- `guildhall.js`：在 `cards[]` 加一張 navCard（如「職業規劃」icon `recruit`/`person`）→ `scene.start('SkillTreeHall')`。
- `main.js`：`scene:[...]` 陣列加入 `SkillTreeHall`。
- `index.html`：加 `<script src="js/scenes/skilltreehall.js?v=11"></script>`（在其他 hall 附近）。

**Stage 2 驗收**：能在大廳進規劃器、選分支排滿 20 級、點數/槽位即時顯示、重置可用、存檔後重開保留。

## Stage 3 — 變身 ＋ 新 handler

### 3-1 變身（大招觸發時的 sprite 切換）
- `js/data.js SPRITES` 新增兩張 12×16 像素圖：`bearman`、`minotaur`（沿用現有調色盤）。
- 變身是 `role:'ultimate'` 的 `type:'transform'` 技能：戰鬥中**大招觸發**時換 sprite ＋掛型態增益，時長到/戰鬥結束還原戰士。**不在出生時固定型態**。
- `js/scenes/battle.js` / `battle.combat.js`：實作 `transform` handler（切 `c.spr` 貼圖、套增益、計時還原）。
- `buildHeroes()`：戰士技能池**直接讀 `SKILLS.warrior`**（無分支解析）。

### 3-2 新 handler（`js/scenes/battle.combat.js` / `battle.js`）
照引擎現有 `getSkill/hasSkillType/skillProc/stun` 模式新增：
- 大招：`transform`（換 sprite＋限時型態增益＋型態強化，計時還原）。
- 被動：`hyperarmor`（免暈＋受傷 ×(1−frac)）、`cover`（後排受傷轉移比例到戰士）、`execute`（對低血敵增傷）。
- 主動：`taunt`（暫拉高被鎖權重）、AoE `stun`（對全體敵呼叫 stun）、`nuke`（單體高倍重擊）、`extraHit`（追加一擊）、`atkBuff`（限時自身 ATK+）。
- 複用既有：`reflect / deathSave / lowHpDef / pierce / critVsStunned / lifesteal / startShield`。

**Stage 3 驗收**：戰士觸發變身大招 → 變熊人/牛頭人、型態增益生效、時長到還原；9 個新 handler（含 `transform`）各自在戰鬥中正確觸發（點角色看明細卡確認）。
