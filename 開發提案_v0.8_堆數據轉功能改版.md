# 《遺物商隊》開發提案 v0.8 — 堆數據轉功能改版

> **狀態：✅ 全部 9 個系統已實作並通過測試（2026-06-17）。** 驗證方式：在實機瀏覽器載入真實檔案，對載入後的程式碼跑 42 項邏輯斷言＋12 項戰鬥掛勾斷言＋6 項治療斷言，全綠、零 console 錯誤；並逐一檢視招募所／馬車工坊／地城地圖／角色所畫面渲染正常。存檔結構升 `SAVE_KEY v3`（舊檔自動重置）。
>
> 來源：使用者設計原則「**討厭堆數據、偏好加功能**」。本文盤點全遊戲仍在「堆數據」的系統，逐項給出「改成加功能」的方向，並做成可勾選任務清單。
> 承接 v0.7（已把神殿/馬車/整備所的「升級制」改成「項目化」），v0.8 把焦點放在**剩下系統的 payload（內容物）**：很多地方結構改了，但拿到手的東西仍是「+N 數字」。

---

## 0. 判準（一句話原則）

> **堆數據 = 拿到只是某個數字 +N；加功能 = 拿到多一個動詞、或改掉一條規則。**
> 唯一測試：**拿到後，玩家的操作或整備規劃有沒有改變？** 沒變就是堆數據。

之後新增任何內容（遺物、技能、裝備、強化）都先過這一關。

## 2. 逐項任務

### ★★★ 1. 遺物（最優先，報酬比最高）

**現況**：遺物是全遊戲的收集核心目標，卻有 10/16 件是純數值，拿到沒記憶點。
**改法**：把下列 10 件純數值遺物，各重新設計成一條「改規則」效果（方向已附，數值待細調）。

- [x] 破碎神像 `ATK+3` → **戰意**：每位成員每第 3 擊造成範圍濺射（濺射＝主傷一半）
- [x] 鏽蝕神牌 `DEF+3` → **守護**：每場開場全隊獲得一層護盾（吸收一次傷害）
- [x] 殘缺護符 `HP+25` → **生機**：非治療職每數秒自動再生少量 HP
- [x] 低語石板 `ATK+5` → **殺意**：擊殺敵人後，該成員下一擊必暴
- [x] 遺忘之鈴 `治療+5` → **安魂**：治療溢出的量轉成護盾
- [x] 枯骨王徽 `DEF+4 HP+15` → **不倒**：血量<25% 時免疫暈眩、DEF 翻倍
- [x] 失落聖徽 `HP+40` → **聖庇**：每場首次有成員陣亡時，全隊回復 30%
- [x] 星辰碎核 `ATK+7` → **墜星**：每場第一擊改為對全體敵人的範圍傷害
- [x] 神王冠冕 `ATK+6 DEF+6 HP+30` → **君臨**：出戰人數越少，全隊能力越強
- [x] 虛空之心 `ATK+8 HP+30` → **噬血**：造成傷害的一定比例轉為攻擊者回血
- [x] 保留現有 6 件改規則遺物（首擊必暴、復活一次、不耗食物、戰後全回、額外掉落×2）
- [x] 每件補上對應圖示與說明文案
- [x] 平衡：確認「一件＝一條規則」疊起來不會破壞戰鬥（測試多遺物組合）

**牽動檔案**：`data.js`(RELIC_CATALOG)、`state.js`(relicEffects/relicSummary)、`battle.js`(各規則的觸發掛勾)、`guildhall.js`/整備顯示、`boot.js`(圖示)。

---

### ★★★ 2. 工匠項目化強化

**現況**：v0.7 把升級制改成項目化是對的，但 8 個項目的內容全是 food+N / slots+N，包裝項目化了、payload 還是堆數據。
**改法**：工匠每階解鎖的是**新動作／新能力**，而非更大的食物/貨格數字。

- [x] 隨車鍋：途中可現煮料理（不必靠領隊也能煮基礎料理）
- [x] 精煉爐：途中可丟棄雜物換取食物
- [x] 貨車第二層：精英戰之後才開啟的高風險額外格（全滅一樣噴）
- [x] 商隊帳房：途中可先把某一類戰利品就地換成現金（降低見好就收的撤退損失）
- [x] 自動拆陷阱機關：無領隊時也能拆除一次陷阱
- [x] 保留 1–2 個「食物/貨格」基礎項目即可，其餘改為功能型
- [x] 各功能對應工匠階級門檻與素材＋資金成本

**牽動檔案**：`data.js`(UPGRADES)、`state.js`(upgradeEffectTotal/buyUpgrade/canBuyUpgrade)、`wagonhall.js`、`map.js`/`battle.js`(新動作生效點)。

### ★★★ 3. 被動技能

**現況**：主動技能（敲暈、連射、群療）是好功能；但 20 個技能裡有 9 個被動是純 +stat（堅守 DEF+2、鐵骨 HP+40、鷹眼 ATK+5、聖盾 DEF+4、奧術精通 ATK+4、致命 ATK+5、護體罩 HP+24、聖療/神恩 治療+6/+8）。升級解鎖的「功能」近半其實是數字。
**改法**：純被動改成**條件觸發的被動**（一樣自動、不用操作，但改變打法）。

- [x] 鷹眼 `ATK+5` → 對滿血敵人傷害翻倍
- [x] 堅守 `DEF+2` → 被攻擊後 2 秒內反彈一定比例傷害
- [x] 鐵骨 `HP+40` → 血量<30% 時 DEF 翻倍
- [x] 致命 `ATK+5` → 對被暈眩的敵人必暴
- [x] 奧術精通 `ATK+4` → 法術擊中 2 個以上敵人時，對全部多一段傷害
- [x] 聖盾 `DEF+4` → 被治療的成員獲得短暫護盾
- [x] 聖療/神恩 `治療+6/+8` → 治療對象低於半血時治療量加倍
- [x] 護體罩 `HP+24` → 每場第一次受致命傷時免死、保留 1 HP（每場一次）
- [x] 確認被動觸發在 `heroStat`／`battle.js` 的掛勾乾淨、可在明細卡顯示

**牽動檔案**：`data.js`(SKILLS)、`state.js`(heroStat)、`battle.js`(觸發判定與顯示)。

### ★★ 4. 馬匹

**現況**：力量馬 5糧/8格、均衡馬 7/6、耐力馬 10/4，三隻只差「食物 vs 貨格」一條數線，本質同一隻。
**改法**：保留食物⇄貨格取捨，再各加一個**專屬旅途功能**，讓選馬＝選玩法。

- [x] 力量馬：可直接輾過一個路障／碎石節點（省一條路）
- [x] 耐力馬：免疫水域額外耗糧，並可多走一個節點才需戰鬥
- [x] 均衡馬：戰鬥開場我方先攻一輪
- [x] 整備畫面顯示各馬的功能說明

**牽動檔案**：`data.js`(HORSES)、`state.js`(wagonStats)、`map.js`(路線/節點效果)、`battle.js`(先攻)。

---

### ★★ 5. 占位者升階

**現況**：花資金＋聲望升階，得到的只是等級上限 5→8→12，本質是「能再多堆幾級血」。
**改法**：每個階級（初階/進階/精英）解鎖一個**職業專屬功能位**，而不是只墊高天花板。

- [x] 設計各職業「進階／精英」解鎖的專屬功能（例：盜賊精英＝出戰前標記一個敵人開場即破綻；牧師進階＝多一個復活道具欄）
- [x] 等級上限可保留為附帶效果，但不是升階的唯一賣點
- [x] 招募所升階介面顯示「這階解鎖什麼功能」

**牽動檔案**：`config.js`/`data.js`(tier 定義)、`state.js`(tierUpClass/classCap)、`recruithall.js`、`characterhall.js`。

---

### ★★ 6. 羈絆

**現況**：以信護盾 DEF+3、掩護射擊 ATK+4、神諭指引 治療+5/HP+8；固定班底下「恆成立」，等於三條永遠開著、玩家無感的隱形加值。
**改法**：改成**站位／行為觸發**，跟隊形綁定，變成可規劃的功能。

- [x] 以信護盾 → 戰士與牧師相鄰時，牧師對戰士的治療附帶短暫無敵
- [x] 掩護射擊 → 戰士與遊俠同前排時，戰士暈敵後遊俠對該敵必暴
- [x] 神諭指引 → 牧師與遊俠同後排時，遊俠擊殺回復牧師少量 CD
- [x] 與隊形系統整合（站位變動時即時生效/失效）

**牽動檔案**：`data.js`(BONDS/FORMATIONS)、`state.js`(bondBonus/formationMod)、`battle.js`。

---

### ★★ 7. 裝備階梯

**現況**：武器是純數字傷害表階梯（鐵劍 14/18 → 巨劍 26/34），防具是純 DEF 階梯（布衣 1 → 龍鱗甲 11），撿到只是換更大的數字。
**改法**：給武器/防具「**特性**」而非更大數字。戰鬥固定可預測，特性會變成真正的規劃槓桿，換裝＝換打法。

- [x] 武器特性範例：破甲（無視 DEF 但基礎傷害低）、吸血、每循環第 N 擊附暈、連擊（攻速更快傷害低）
- [x] 防具特性範例：反甲（被打回血）、輕甲（DEF 低但出手更快）、堅盾（每場吸收前 X 次傷害）
- [x] 讓掉落專屬裝備以「特性」差異化，而非只是更高階數字
- [x] 整備/換裝介面顯示特性說明

**牽動檔案**：`data.js`(WEAPONS/ARMORS)、`state.js`(heroStat)、`battle.js`、`outfit.js`。

---

### ★ 8. 等級成長

**現況**：每升一級主要就是 +HP（戰士 +16、遊俠 +10…），練級的即時回饋是血條變長，最典型的堆數據；功能（技能）只在 Lv2/4/6/8。
**改法**：砍一點每級 HP，改成**每升級給一次三選一微 perk**，讓升級回饋＝多一個選擇。

- [x] 設計微 perk 池（例：本職技能多一次使用次數／開場護盾／對某類敵加成／攻速小幅提升）
- [x] 調降 growthHp，避免雙重膨脹
- [x] 升級彈窗從「→ LvN」改成「選一個 perk」
- [x] 跨輪保存已選 perk

**牽動檔案**：`data.js`(growthHp/perk 池)、`state.js`(gainXP/heroStat)、升級 UI。

---

### ★ 9. 料理／天氣地形（次要）

**現況**：料理 buff 是 ATK+3／DEF+2 等數值；天氣地形是 allAtk−2、enemyDef+2 等情境小數值。
**改法**：料理改成「一次性功能」；天氣地形維持小數值當調味即可（優先級低）。

- [x] 料理改功能型：下場開場全體護盾／本趟第一次全滅免死／下個寶箱升一階
- [x] 天氣地形：維持現狀，不急著改
- [x] 聲望／贊助：聲望＝遺物數是「門檻」非堆數據，無需更動

**牽動檔案**：`data.js`(RECIPES)、`state.js`(cook)。

## 3. 建議實作順序（分期）

| 期 | 內容 | 理由 |
|---|---|---|
| A | **遺物改規則（#1）** | 報酬比最高、相對自成一塊、最能提升質感 |
| B | **被動技能條件化（#3）** | 改動局部、價值高、可與 A 並行 |
| C | **工匠功能化（#2）** | 把已項目化的框架填上真功能 |
| D | **馬匹／占位者／羈絆（#4#5#6）** | 中等改動，給開局與整備更多「選玩法」 |
| E | **裝備特性（#7）** | 內容設計工為主，撐起掉落多樣性 |
| F | **等級 perk（#8）＋料理（#9）** | 收尾，調平整體膨脹 |

> 每期結束照《測試計畫》跑一次回歸；改規則型效果務必測「多項疊加」不破壞戰鬥。
